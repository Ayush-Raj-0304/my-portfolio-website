## üìò What is MDX?

MDX (Markdown for the component era) is a file format that lets you seamlessly write JSX within Markdown files. It combines the simplicity of Markdown with the expressiveness of React components.

In essence, MDX allows developers to:

- Write Markdown like usual (`#`, `##`, `**bold**`, etc.)
- Import and use React components inside `.mdx` files
- Include dynamic or interactive elements in documentation or blogs

```jsx
import Chart from './components/Chart';

# My Dashboard

Here's a dynamic chart:
<Chart data={[1, 2, 3]} />
```

## üöÄ Why Use MDX?

MDX shines in scenarios where you want content to be expressive, interactive, and component-driven. Some use cases include:

- Technical blog posts with live React components
- Developer documentation with code playgrounds
- Custom-styled content blocks
- Interactive tutorials and demos

### Benefits

- **Component-based content**: Use your existing components directly.
- **Reusable content**: Share UI elements between docs and app.
- **Full JavaScript power**: Add logic, fetch data, and render conditionally.

## üõ†Ô∏è Getting Started

To start using MDX, choose a compatible framework like Next.js, Astro, or Gatsby. Here's how you can set it up in **Next.js**:

1. Install the required packages:

```bash
npm install @mdx-js/loader @mdx-js/react
```

2. Update your Next.js config (`next.config.js`):

```js
const withMDX = require('@next/mdx')();
module.exports = withMDX({
  pageExtensions: ['js', 'jsx', 'mdx']
});
```

3. Create an `.mdx` file:

```mdx
---
title: Hello MDX
---

# Welcome to MDX!

<CustomButton>Click Me</CustomButton>
```

## üîç Example: Fetching Data

You can even fetch data inside your MDX files:

```js
export async function getSource() {
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();
  return data;
}
```

## üß© MDX Components Mapping

To map custom components in MDX, wrap your MDX provider:

```js
import { MDXProvider } from '@mdx-js/react';
import CodeBlock from './CodeBlock';

const components = {
  pre: CodeBlock,
};

<MDXProvider components={components}>
  <MyMDXContent />
</MDXProvider>
```
